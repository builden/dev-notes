# 经典 23 种设计模式

设计模式主要是为了应对代码的复杂性，让其满足开闭原则，提高代码的扩展性

### 设计模式要做的事情本质就是`解耦`

- 创建型模式是将创建和使用代码解耦
- 结构型模式是将不同功能代码解耦
- 行为型模式是将不同的行为代码解耦

## 创建型 (Creational Patterns) -- 5 种

- 主要解决`对象的创建`问题

## 结构型 (Structural Patterns) -- 7 种

- 主要解决`类或对象的组合或组装`问题

## 行为型 (Behavioral Patterns) -- 11 种

- 主要解决`类或对象之前的交互`问题

1. 观察者模式 (Observer Pattern)
   > 是将观察者和被观察者的代码解耦
